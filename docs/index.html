Base templates

Every FlowNet workflow is built from two kinds of blocks: triggers, which start a run, and steps, which do the work. These are the built-in templates that ship with the core and ai integrations — available in every workflow, no setup required.

Triggers

A workflow starts with exactly one trigger. It defines the event that kicks off a run and the data available to the rest of the workflow.

Manual Trigger

manual

Runs a workflow on demand, straight from the FlowNet editor. No external event required.

Webhook

webhook

Fires a workflow whenever FlowNet receives an HTTP request on the workflow's unique webhook URL.

Scheduled

cron

Runs a workflow on a recurring schedule using a standard cron expression, with ready-made presets.

Shopify

shopify

Fires a workflow from Shopify store events — new products, updates, inventory changes, and more.

Steps

Steps are chained after a trigger to process data, call external services, branch logic, or run custom code.

Debug

debug

Outputs the current state or specified variables to the run log, for inspecting a workflow mid-flight.

HTTP Request

http_call

Makes an HTTP request to any URL with a configurable method, query params, headers, and body.

Lua script

lua_script

Executes a custom Lua script with access to workflow variables, for logic no built-in step covers.

Condition

condition

Evaluates a boolean expression and branches the workflow down a true or false path.

Loop

loop

Iterates over a list, running the downstream branch once per item.

Shopify GraphQL

shopify_graphql

Executes any Shopify Admin GraphQL query or mutation directly against the connected store.

Gemini

gemini

Generates text with Google Gemini — summarize, classify, or draft content mid-automation.
