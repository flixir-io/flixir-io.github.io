Frequently asked questions

What is Flixir?

Flixir is a visual workflow builder for Shopify. You connect a trigger (an order, a customer, a stock change) to a chain of steps on a canvas, and the engine runs it whenever the trigger fires — no app code to deploy.

Do I need to know how to code?

No. Building a workflow is drag-and-drop: place steps on the canvas and wire them together. Advanced steps let you reference data from earlier steps with a simple expression syntax, but that’s optional, not required.

Which Shopify events can trigger a workflow?

Order, customer, and inventory events are supported today, backed by Shopify’s native webhooks and GraphQL Admin API. See the docs for the full, current list of triggers and actions.

Is my store data secure?

Workflows run against your Shopify store using Shopify’s own authenticated APIs. We don’t sell or share store data, and each workflow only accesses the scopes it needs.

When can I start using it?

Flixir is in active development. Follow the GitHub repository for progress, and check the docs for what’s already built.
